# Pitch Booking Flutter Client

واجهة Flutter بسيطة للتكامل مع خادم الحجز الموجود داخل مجلد `backend/`. التطبيق يعرض قائمة الملاعب مع إمكانية البحث وحجز موعد متاح بعد التأكد من عدم وجود حجوزات سابقة لنفس التوقيت.

## المتطلبات

- تثبيت Flutter SDK (إصدار 3.13 أو أحدث موصى به).
- تشغيل خادم الـ backend الموجود في `backend/` على نفس الجهاز أو تعديل عنوان الـ API.

## الإعداد والتشغيل

```bash
cd frontend
flutter pub get
# في المرة الأولى يمكنك توليد ملفات المنصات بالأمر التالي إذا لزم الأمر:
# flutter create .
flutter run # أو flutter run -d chrome لتجربة نسخة الويب
```

### تغيير عنوان الـ API

افتراضيًا يتواصل التطبيق مع `http://localhost:4000`. يمكن تعديل ذلك بطريقتين:

1. تمرير قيمة متغير البيئة `API_BASE_URL` عند التشغيل:
   ```bash
   flutter run --dart-define=API_BASE_URL=http://192.168.1.10:4000
   ```
2. تعديل القيمة الافتراضية داخل `ApiClient` في `lib/services/api_client.dart`.

## المزايا الحالية

- عرض الملاعب المتاحة مع تفاصيل السعر، نوع الأرضية والمميزات.
- بحث لحظي عن الملاعب بالاسم أو الموقع.
- نموذج حجز يتحقق من البيانات ويعرض المواعيد المحجوزة مسبقًا.
- تفعيل/تعطيل الوضع الليلي بسهولة من التطبيق.

## أفكار للتطوير القادم

- إضافة شاشة لعرض حجوزات المستخدم الحالية مع إمكانية الإلغاء.
- دعم رفع صور للملاعب وعرضها داخل القائمة.
- إضافة طبقة تخزين محلي (Hive مثلًا) لعرض البيانات في حال انقطاع الإنترنت.
- ربط النظام بإشعارات Push أو بريد إلكتروني بعد تأكيد الحجز من الأدمن.
